# Templates for @voltrix/decorator

This directory contains reusable templates and code patterns for developing with the @voltrix/decorator package.

## Template Categories

### 🏗️ Decorator Templates
- **extensible-decorator.md** - Template for creating new extensible decorators
- **validation-decorator.md** - Template for parameter validation decorators  
- **security-decorator.md** - Template for security and authorization decorators
- **middleware-decorator.md** - Template for middleware and processing decorators

### 🎯 Usage Templates
- **controller-class.md** - Template for creating decorated controller classes
- **service-class.md** - Template for injectable service classes
- **application-setup.md** - Template for @VoltrixApp application configuration
- **integration-examples.md** - Common integration patterns and examples

### 🧪 Testing Templates  
- **decorator-test.md** - Template for testing custom decorators
- **integration-test.md** - Template for testing decorator integration
- **performance-test.md** - Template for performance testing decorators
- **mock-setup.md** - Template for test mocking and setup

### 📚 Documentation Templates
- **decorator-documentation.md** - Template for documenting new decorators
- **api-documentation.md** - Template for API endpoint documentation
- **changelog-entry.md** - Template for changelog entries
- **migration-guide.md** - Template for version migration guides

## Usage Guidelines

### For Decorator Developers
1. Use **extensible-decorator.md** as starting point for new decorators
2. Follow patterns in **validation-decorator.md** for parameter handling
3. Reference **security-decorator.md** for authorization patterns
4. Apply **middleware-decorator.md** for processing logic

### For Application Developers
1. Start with **controller-class.md** for API endpoints
2. Use **service-class.md** for business logic components
3. Configure with **application-setup.md** for app initialization
4. Reference **integration-examples.md** for common patterns

### For Testing
1. Follow **decorator-test.md** for unit testing decorators
2. Use **integration-test.md** for full system tests
3. Apply **performance-test.md** for benchmarking
4. Setup with **mock-setup.md** for test isolation
