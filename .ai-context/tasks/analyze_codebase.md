# Task 4: Analyze Codebase

## Objective
Perform a deep analysis of the existing codebase to understand architecture, patterns, dependencies, and identify areas for improvement.

## Analysis Areas

### 1. Architecture Analysis
- Module structure and dependencies
- Design patterns implementation
- Separation of concerns
- API design and interfaces

### 2. Code Quality Metrics
- Complexity analysis
- Code duplication detection
- Maintainability index
- Technical debt assessment

### 3. Performance Analysis
- Runtime performance bottlenecks
- Memory usage patterns
- Database query optimization
- API response time analysis

### 4. Security Review
- Input validation patterns
- Authentication/authorization implementation
- Data sanitization practices
- Vulnerability scanning results

## Success Criteria
- [ ] Architecture documentation created
- [ ] Code quality report generated
- [ ] Performance recommendations provided
- [ ] Security assessment completed

## Expected Output
Comprehensive codebase analysis report saved as `knowledge/codebase-analysis.md`
